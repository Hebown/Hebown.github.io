Data-Visualization-Note

本文是博主短学期课程的笔记博客。主要用来梳理一下下自己学习的思路。

这个课程是双语教学，还是有点抽象的。

第一节课：#

basic:#

  • 陈为：
    • 简要介绍了一下历史，成员什么的。
    • 还讲了下面的课程成绩组成：
      • project1（20%）：
        • 个人学习可视化软件的报告；
      • project2（50%）：
        • 三人一组，做一个大的可视化实验报告。
  • 应该是助教，不过是哪一个我就不知了：
    • 介绍了大数据可视化的含义:将复杂的数据转化成直观可以感受的可视化结果。
    • 还给出了一些其他的应用，不仅应用在静态的数据分析之上，而且能够应用在动态的数据分析之中。
      • 根本上，大数据的目的就是改变数据展示的方式，便于分析。
      • 给出了一个研究上的结果：大约有一半的人在被问路的时候不关心是谁问的，甚至换了人也看不出来。这是我们的认知局限。
      • 进一步，从数据可视化产生了数据分析的可视化。
      • 然后介绍了现代的数据可视化工具，例如这个课程使用的tableau，和powerbi、Google’s Visualization Team等等。
      • 以及数据挖掘，简单来说就是开挂找信息。
      • VIS也是比较常见的应用，就是给企业设计视觉上的形象。

lecture:#

我自己认为，所谓的lecture是请一些人做点报告什么的，这个内容就当是拓展自己的视野，不作为考试的要求。

  • 加州海归老师：
    • 主题：Visualization as the pathway to human-ai teaming.
      1. what is human-ai teaming?
        • 我的感觉就是人机合作。用人工智能的力量增强人的能力。
          • 比如生物识别、gpt、ai艺术（各种各样的绘画）、还有之前见的ai治病。
          • 目前ai已经在很多领域大展拳脚，不过，在ai的进一步应用上，还有路要走。
        • 一个基本的结构就是，人类给出期望和思路，让ai进一步处理这些需求，并返回可能的结果作为进一步分析的铺垫。
          • 一般来说，ai善于解决涉及到繁复操作的问题，很难利用已有知识来解决新的问题；而人类善于创作，可以解决新问题。这是我们所了解的基本内容。
      2. data visualization:
        • 然而我们想要使用ai，就要解决ai与人类沟通之间的问题（visual interface），应该将ai生成的结果转化为人类便于理解的、富有启发性的内容。比如表格数据转化为图表数据就是一种可视化，相信大家还见过一个，世界地图上亮起的灯，灯光越亮的地方就是城市越集中的地方，这也可以视为是一种可视化。
        • 信息爆炸已然是一个已经烂了的词汇，我们能够从中获得的“信息”就是，未来，如何高效处理信息将成为重中之重。
      3. monitor:
        • 高效训练模型需要处理异常内容，一般的流程就是：人工设定异常内容，给出修改意见，然后交给计算机自己来根据需求进行变化。
        • 后面讲的就比较抽象了，上课没看明白。估计是给我前面的博士生讲的。
      4. examples:
        • 讲了一些具体情形中的流程的例子

数据分析/处理：#

  • data management:
    • data profiling:
      • data overview:
        • data domain:数据属于什么领域；
        • 数据规模；
        • 数据类型；
        • 数据范围；
      • data quality:
        • 缺失信息；
        • 非法信息；
        • 冗余信息；
        • 同质信息：比如zju和浙江大学本质上是一样的。
  • data wrangling（整理）:对数据做出若干操作，比如排序/提取/次数据…
    • 有一个比较抽象的join操作，分为inner，right/left，full，其实就是将两个不同表中的相同元素组合起来成为一个表，不过有不同的组合策略；
